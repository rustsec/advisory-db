# Unsafe removal of environment variables

Affected versions of this crate expose unsetting environment variables in safe functions. Since unsetting environment variables is thread-unsafe on Linux (see [documentation of `std::env::remove_var`](https://doc.rust-lang.org/std/env/fn.remove_var.html)), this is unsound.

There's a discussion of this issue at [sd-notify#9](https://github.com/lnicola/sd-notify/issues/9) with fix introducing new APIs at [sd-notify#13](https://github.com/lnicola/sd-notify/pull/13).
