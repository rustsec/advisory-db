# Out-of-bounds access due to pointer arithmetic bug in `iter()` and `iter_mut()`

The `iter()` and `iter_mut()` APIs compute
`current = (&children[0] as *const *const RawAutoChild).sub(1)`, which
performs a pointer subtraction that returns a result before the start of the
allocation. This is immediate undefined behavior as per the safety contract of
the `sub` method on pointers, and subsequently results in out-of-bounds access.

This can be triggered without the use of unsafe code via the `iter()` and
`iter_mut()` methods on `AutoVec`.
