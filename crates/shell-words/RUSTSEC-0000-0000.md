# `~` not escaped (shell_words::quote)

Affected versions of this crate incorrectly looked for `˜` (U+02DC SMALL TILDE) instead of the intended `~` (U+007E TILDE)

This could lead to exposing the users home path, or in certain cases path traversal. 

This was corrected in commit efe8162, included in version `1.1.1`.
