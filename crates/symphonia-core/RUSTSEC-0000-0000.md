# Panic in `TimeBase::new` on zero-valued timescale in `symphonia-core`

Affected versions of this crate panic in `TimeBase::new` when constructed with a zero numerator or denominator. Symphonia demuxers pass container metadata into this constructor, so an attacker supplying a crafted audio file can use this to trigger a denial of service in any application that parses untrusted input with Symphonia.

The flaw was corrected upstream by changing `TimeBase::new` to accept `NonZero<u32>` arguments. The fix is included in `symphonia-core 0.6.0-alpha.1` and later.
