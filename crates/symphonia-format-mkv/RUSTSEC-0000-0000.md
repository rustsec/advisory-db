# Panic on out-of-range MKV timestamp scale in `symphonia-format-mkv`

The MKV demuxer unwraps a narrowing conversion on the `TimestampScale` field read from container metadata, causing a panic when the value exceeds `u32::MAX`. An attacker supplying a crafted Matroska/WebM file can use this to trigger a denial of service in any application that parses untrusted input with Symphonia.

The flaw was corrected upstream in `symphonia-format-mkv 0.6.0-alpha.1`.
