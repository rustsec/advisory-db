# Reachable `assert_eq!` in MKV EBML iterator in `symphonia-format-mkv`

The MKV EBML iterator contains an `assert_eq!` that is reachable when child element boundaries are inconsistent with their parent's declared size, causing a panic. An attacker supplying a crafted Matroska/WebM file can use this to trigger a denial of service in any application that parses untrusted input with Symphonia.

The flaw was corrected upstream by rewriting the EBML iterator; the new implementation returns a decode error instead of asserting. The fix is included in `symphonia-format-mkv 0.6.0-alpha.1` and later.
